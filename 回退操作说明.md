## 手动操作步骤

### 方法1：使用PowerShell脚本（推荐）

1. **关闭所有正在运行的git/less进程**
   - 在任务管理器中结束所有 `less` 和 `git` 相关进程

2. **打开新的PowerShell窗口**
   - 导航到项目目录：
   ```powershell
   cd "D:\HuaweiMoveData\Users\Tang\Desktop\平高项目AI"
   ```

3. **执行脚本**
   ```powershell
   powershell -ExecutionPolicy Bypass -File reset_to_noon.ps1
   ```

### 方法2：手动执行命令

1. **打开新的PowerShell窗口**

2. **导航到项目目录**
   ```powershell
   cd "D:\HuaweiMoveData\Users\Tang\Desktop\平高项目AI"
   ```

3. **禁用分页器**
   ```powershell
   $env:GIT_PAGER = ""
   git config --global core.pager ""
   ```

4. **查看当前提交（找到中午12点的提交）**
   ```powershell
   git log --oneline -20
   ```

5. **回退到指定提交**
   
   根据提交历史，中午12点应该在以下提交之前：
   - `30ef8af` - 重新设计项目列表（这是下午的提交）
   
   回退3个提交（回到中午12点之前）：
   ```powershell
   git reset --hard HEAD~3
   ```
   
   或者回退到特定提交：
   ```powershell
   git reset --hard af55c38
   ```

6. **验证回退结果**
   ```powershell
   git log --oneline -5
   ```

7. **强制推送到GitHub**
   ```powershell
   git push origin main --force
   ```

## ⚠️ 重要提示

- **强制推送会覆盖远程仓库的历史**，请确保这是您想要的操作
- 如果团队中有其他人也在使用这个仓库，请先通知他们
- 建议在执行前先备份当前代码

## 如果需要保留某些更改

如果您想保留某些文件，可以使用：
```powershell
# 软回退（保留工作区更改）
git reset --soft HEAD~3

# 然后选择性地提交某些文件
git reset HEAD <不需要的文件>
git commit -m "保留某些更改"
```
